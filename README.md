An immersive, text-based sci-fi survival game and biometric synchronization tracker. Step into the shoes of a commander managing the life support systems of humanity's last hope: the Space Ark. 

Your mission objective? Sync your body with the Ark's core systems by logging daily hydration and physical overdrive tasks to ensure the journey to our New Home succeeds.

## 🎮 Game Features & Mechanics

* **Atmospheric Sci-Fi HUD:** Features a stunning, animated terminal interface with neon glowing themes (`Orbitron` & `Space Mono` fonts) and a dynamic, flickering starfield background.
* **Daily Synchronization Protocols:** Navigate through consecutive days of logs, unlocking new content as you complete real-life physical calibration checkpoints (tracking water levels and workout repetitions).
* **Dynamic Progress Tracking:** Visual HUD gauges update in real-time to show your current Water levels and Thruster energy statuses.
* **Persistent Save States:** Powered by Browser LocalStorage. Your journey, unlocked days, and biometric logs are saved automatically, meaning you can close the browser and resume your mission right where you left off.
* **Multi-Language Support (Localization):** Fully playable in three languages: English (**EN**), Russian (**RU**), and Serbian (**SR**).
* **Safe Protocol Reset:** Features a full memory wipe option (`Reset Progress`) to reload the matrix and start a completely new campaign.

## 🛠️ Under the Hood (Tech Stack)

This project is optimized to run lightning-fast using completely vanilla, client-side technologies:

* **HTML5:** Semantic architecture, text terminal layouts, and responsive controls.
* **CSS3 (Custom Variables & Animations):** Custom cyber-themes (`--neon-cyan`, `--neon-pink`), starfield rendering, adaptive mobile responsiveness, and custom terminal scrollbars.
* **Vanilla JavaScript (ES6+):** Complete state machine management, multi-lingual dictionary routing, user event handling, mathematical scaling for HUD progression bars, and LocalStorage integration.

## 🕹️ Mission Initialization (How to Run Locally)

Since the simulation runs strictly on the client side, there is zero configuration or server overhead required.

1. **Acquire Files:** Clone this repository or download and extract the source ZIP file.
2. **Boot the System:** Simply locate `index.html` and double-click to launch it instantly in any modern web browser.
